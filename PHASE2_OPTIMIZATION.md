## 第二阶段优化重点

### 1. 图片优化（高优先级）

#### 问题
- `shine.png` (9.9M) - 背景装饰图
- `video_dark_bak.png` (4.6M) - 视频预览图
- `bg.png` (2.0M) - 背景图
- 多个 ability 图片 (1-2M)

#### 解决方案
1. **压缩现有图片**
   - 使用 TinyPNG/ImageOptim 压缩
   - 目标：减少 70-80% 文件大小

2. **转换为 WebP 格式**
   - 创建 WebP 版本
   - 使用 Next.js Image 组件自动优化
   - 保留 PNG 作为 fallback

3. **添加 alt 文本**
   - 所有图片添加描述性 alt
   - 提升可访问性和 SEO

4. **懒加载**
   - 非首屏图片使用 loading="lazy"
   - 优化 LCP (Largest Contentful Paint)

### 2. 内部链接优化（中优先级）

#### 当前状态
- 导航栏：Docs, Pricing
- Footer：Terms, Privacy
- 缺少：相关内容推荐、面包屑导航

#### 改进方案

   - Enterprise 页面：推荐 Pricing、Docs
   - FAQ 页面：推荐相关 FAQ 分类

2. **面包屑导航**
   - 所有子页面添加面包屑

3. **锚点链接**
   - FAQ 页面添加目录跳转
   - 长页面添加"返回顶部"

### 4. 性能优化（高优先级）

#### 目标
- Lighthouse Performance: >90
- LCP: <2.5s
- FID: <100ms
- CLS: <0.1

#### 改进方案
1. **代码分割**
   - 动态导入非关键组件
   - 减少首屏 JS bundle 大小

2. **字体优化**
   - 使用 `font-display: swap`
   - 预加载关键字体
   - 考虑使用系统字体

3. **CSS 优化**
   - 移除未使用的 CSS
   - 内联关键 CSS
   - 延迟加载非关键 CSS

4. **第三方脚本优化**
   - 延迟加载分析脚本
   - 使用 Partytown 隔离第三方脚本
